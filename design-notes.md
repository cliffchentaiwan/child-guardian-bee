# 設計文件資料來源分析

根據 PDF 設計文件，「案例資料庫（風險揭露區）」應彙整以下資訊來源：

## 資料來源類型
1. **媒體報導**（含地方新聞）
2. **政府公告**（如法院、社福機構、性侵加害人名冊）
3. **社群輿情**（例如高熱度社群討論貼文，但標註「未證實」）

## 每則紀錄包含
- 案件類型標籤（性騷擾、偷拍、情緒虐待等）
- 發生地區與時間
- 加害人身分概略（例如：「30歲男性，曾任才藝班老師」）
- 來源連結（媒體、公告）

目前版本為「靜態前端」(web-static)，無法直接串接外部 API。
若要串接真實政府資料庫，需要：
1. 升級為 web-db-user（全端專案）
2. 建立後端 API 代理
3. 處理 CORS 和 API 認證
