# 台灣政府兒少保護相關資料庫 API 可用性研究報告

## 研究日期：2026年1月15日

---

## 一、已確認的政府資料來源

### 1. 衛福部醫事人員性別事件資訊專區 ✅ 公開可查
- **網址**：https://ma.mohw.gov.tw/Accessibility/VIOSearch/MASearchVIO
- **功能**：查詢醫事人員（醫師、牙醫師、護理人員等）是否有性別事件紀錄
- **資料範圍**：112年性平三法修正後的公開資訊
- **限制**：
  - 需輸入驗證碼（CAPTCHA）
  - 無公開 API
  - 僅限醫事人員，不包含教師、保母等
- **可行性**：可透過後端爬蟲定期抓取，但需處理驗證碼

### 2. 教育部不適任人員通報及查詢系統 ❌ 不公開
- **網址**：https://unfitinfo.moe.gov.tw/
- **功能**：查詢不適任教育人員
- **限制**：
  - 僅供教育單位使用，需帳號密碼登入
  - 一般民眾無法使用
  - 無公開 API
- **可行性**：無法直接串接

### 3. 司法院裁判書開放 API ✅ 有公開 API
- **網址**：https://opendata.judicial.gov.tw/
- **功能**：查詢法院裁判書全文
- **資料範圍**：所有公開的法院判決書
- **API 說明**：
  - 需申請會員帳號
  - 提供 RESTful API
  - 可取得裁判書異動清單及全文
- **可行性**：✅ 可串接，需申請帳號

### 4. 政府開放資料平台 (data.gov.tw) ✅ 有統計資料
- **網址**：https://data.gov.tw/
- **可用資料集**：
  - 性侵害案件統計資料（僅統計數據，無個案資料）
  - 兒少保護統計資料
- **限制**：僅提供統計數據，不提供個案查詢
- **可行性**：可用於顯示趨勢統計，但無法用於個人查詢

### 5. 性侵害犯罪加害人登記資料 ❌ 不公開
- **法規**：性侵害犯罪加害人登記報到查訪及查閱辦法
- **限制**：
  - 僅限特定機關（學校、機構）申請查閱
  - 需提供被查閱人姓名、身分證字號、出生年月日
  - 一般民眾無法查詢
- **可行性**：無法直接串接

---

## 二、可行的技術方案

### 方案 A：整合司法院裁判書 API（推薦）
**優點**：
- 官方公開 API，合法使用
- 資料完整，包含所有公開判決
- 可搜尋性侵害、兒虐相關案件

**實作方式**：
1. 申請司法院資料開放平台會員
2. 使用 API 定期抓取相關案件判決書
3. 建立本地資料庫，進行姓名模糊比對
4. 自動遮罩姓名中間字

**限制**：
- 需申請帳號（約1-3工作天）
- 僅限已公開判決，不包含偵查中案件

### 方案 B：新聞媒體爬蟲 + 人工審核
**優點**：
- 可獲取最新案件資訊
- 涵蓋範圍廣

**實作方式**：
1. 定期爬取主要新聞網站（自由時報、聯合報、中時等）
2. 使用關鍵字篩選（性侵、兒虐、狼師等）
3. 人工審核後入庫

**限制**：
- 法律風險：新聞內容可能有誤報
- 需標註「未經司法認定」
- 需處理著作權問題

### 方案 C：社群輿情監控
**優點**：
- 可獲取第一手爆料資訊

**實作方式**：
1. 監控 PTT、Dcard 等社群平台
2. 關鍵字篩選相關討論
3. 人工審核，標註「未證實」

**限制**：
- 資訊可靠性低
- 法律風險高（誹謗、個資法）
- 必須標註「未經證實」

---

## 三、法律與倫理考量

### 個人資料保護法
- 公開個人犯罪紀錄需符合「增進公共利益所必要」
- 建議：僅顯示已公開判決資訊，遮罩部分姓名

### 誹謗風險
- 未經證實的資訊可能構成誹謗
- 建議：提供澄清機制，讓被標註者可申請移除

### 免責聲明
- 必須明確標示「資料僅供參考，非絕對比對」
- 建議使用者自行查證

---

## 四、建議的實作優先順序

1. **第一階段**：串接司法院裁判書 API
   - 申請帳號
   - 建立後端 API 代理
   - 實作姓名模糊比對

2. **第二階段**：整合衛福部醫事人員查詢
   - 建立爬蟲（需處理驗證碼）
   - 或提供外連引導使用者自行查詢

3. **第三階段**：新聞媒體整合（可選）
   - 建立新聞爬蟲
   - 人工審核機制

---

## 五、結論

目前台灣政府的兒少保護相關資料庫，**大多數不對一般民眾公開**。唯一可直接串接的是**司法院裁判書 API**，可用於查詢已公開的法院判決。

若要建立真正可用的查詢平台，建議：
1. 升級為全端專案（web-db-user）
2. 申請司法院 API 帳號
3. 建立後端資料庫儲存處理後的資料
4. 實作模糊比對搜尋引擎
5. 加入適當的法律免責聲明
