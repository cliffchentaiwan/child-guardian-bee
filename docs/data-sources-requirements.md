# 兒少守護小蜂 - 資料來源需求分析

根據設計文件，系統需要整合以下資料來源：

## 案例資料庫（風險揭露區）

需彙整下列資訊來源，半結構化整理與標記地區與角色（如：學校老師、才藝老師、保母、陪玩人員）：

### 1. 媒體報導（含地方新聞）
- **目前狀態**: ✅ 已整合
- **來源**: 中央社、自由時報 RSS
- **同步頻率**: 可 24 小時

### 2. 政府公告
需整合以下政府資料庫：

#### 2.1 司法院裁判書系統
- **目前狀態**: ✅ 已整合
- **API**: https://data.judicial.gov.tw/jdg/api/
- **服務時間**: 每日 0:00-6:00
- **內容**: 法院判決書

#### 2.2 社福機構相關
- **目前狀態**: ❌ 尚未整合
- **可能來源**:
  - 衛福部社會及家庭署
  - 各縣市社會局公告
  - 托嬰中心/幼兒園裁罰紀錄

#### 2.3 性侵加害人名冊
- **目前狀態**: ❌ 尚未整合
- **說明**: 此為敏感資料，需確認是否有公開 API 或查詢機制
- **可能來源**: 
  - 衛福部性侵害犯罪加害人登記報到查詢系統
  - 需確認是否開放 API

### 3. 社群輿情
- **目前狀態**: ⚠️ 部分整合（標註為「未證實」）
- **說明**: 高熱度社群討論貼文，但標註「未證實」
- **可能來源**:
  - PTT
  - Dcard
  - Facebook 社團

## 每則紀錄包含
- 案件類型標籤（性騷擾、偷拍、情緒虐待等）
- 發生地區與時間
- 加害人身分概略（例如：「30歲男性，曾任才藝班老師」）
- 來源連結（媒體、公告）
